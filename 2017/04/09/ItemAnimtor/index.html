1.1 SimpleItemAnimator的分析

1.1.1 Public methods

  1. animateAdd(RecyclerView.ViewHolder holder).
    • Called when an item is added to the RecyclerView.
  2. animteAppearance(RecyclerView.ViewHolder viewholder, RecyclerView.ItemAnimator.ItemHolderInfo preLayoutInfo, RecyclerView.ItemAnimator.ItemHolderinfo postLayoutInfo)
    • Called by the RecyclerView when a ViewHolder is added to the layout.
  3. animateChange(RecyclerView.ViewHolder oldHolder, RecyclerView.ViewHolder newHolder, RecyclerView.ItemANimator.ItemHolderInfo preInfor, RecyclerView.ItemAnimator.ItemHolderInfo postInfo)
    • RecyclerView call when an adapter item 一直存在和 RecyclerView has received a notifyItemChanged(int) call for it.
  4. animateDisapperance()
    • a ViewHolder disappeared for the layout
  5. animateMove()
    • an item 移进 RecyclerView.
  6. animatePersistence()
    • a ViewHodler is always here, the RecyclerView don't call notifyitemchanged(int) for it or a notifyDataSetChanged() call.
  7. animateRemove
    • when an item is removed from the RecyclerView
  8. canReuseUpdatedVieHolder()
    • an Item is changed, 是否重用 同一个ViewHolder or RecyclerView 应该create a copy of the item and ItemAnimator will use both to run the animation

  14. dispahtMoveStart

  17. getSupportsChangeAnimations()
    • whether this ItemAnimator suports animations of change event.
  18. onAddFinished(RecyclerView.ViewHolder item)
    • an add animation has ended, default is nothong
  19. onAddStarting(RecyclerView.ViewHolder item)
    • is starting
